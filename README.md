# Volkswagen 意图评估系统

意图分析结果的自动评估工具，用于比较 XML 标注和 OSC 输出的匹配程度。

## 项目结构

```
volkswagen_intention_eval/
├── 1_score_calculator.py      # 核心计分脚本
├── 2_visualize_scores.py      # 得分可视化
├── 0_osc_intention_eval.py    # OSC 意图分析
├── 0_xml_intention_eval.py    # XML 意图解析
├── data/                      # 数据目录
│   ├── CQU/                   # CQU 场景数据
│   └── 02_CIDAS场景/          # CIDAS 场景数据
└── cqu_evaluation_results.json # 评估结果
```

## 数据层级结构

评估数据分为 5 个层级：

| 层级 | 名称 | 说明 |
|------|------|------|
| 第1层 | 基本信息层 | 主车数量、NPC数量、行人数量等 |
| 第2层 | 场景环境层 | 道路类型、拓扑、车道、天气等 |
| 第3层 | 参与者信息层 | 所有参与者（主车/NPC/行人）详细信息 |
| 第4层 | 行为语义层 | 触发条件、时序关系、终止条件等 |
| 第5层 | 意图推理层 | 驾驶意图、冲突类型、风险等级等 |

## 计分规则

### 第三层（参与者信息层）
- 采用**参与者整体匹配**策略
- 将第四层的动作序列（主车动作序列、他车他者动作序列）合并到对应参与者中
- 通过大模型判断两个参与者是否描述同一个实体（忽略 ID 命名差异）
- 匹配后进行整体内容一致性判断

### 其他层级
- 采用**逐项匹配**策略
- 内容完全一致直接得分
- 不一致时由大模型判断语义是否相同
- "未涉及"项不参与计分

## 使用方法

### 运行评估

```bash
python 1_score_calculator.py
```

### 可视化结果

```bash
python 2_visualize_scores.py
```

## 评估结果

评估结果保存在 `cqu_evaluation_results.json`，包含：
- 每个文件的匹配得分
- 详细处理日志
- 汇总统计数据
